# Glossario mestre

Este documento define como o produto deve nomear suas experiencias principais.

O objetivo e reduzir drift entre telas, analytics, erros e futuras campanhas.

## Regras gerais

- preferir portugues consistente
- evitar mistura desnecessaria com ingles
- preferir linguagem reconhecivel pelo usuario
- manter termos de dominio apenas quando realmente ajudarem

## Termos oficiais do produto

### Home

Usar:

- `Home`

Evitar:

- dashboard
- tela principal

### Jornal

Usar:

- `Jornal`

Evitar:

- journal

Observacao:

- se o nome em ingles for importante para branding depois, ele deve virar decisao de marca global, nao aparecer misturado

### SOS

Usar:

- `SOS`

Uso secundario permitido:

- `momento critico`
- `suporte imediato`

Evitar como nome principal:

- panico
- botao de emergencia

Observacao:

- `Panico` pode existir como atalho interno temporario, mas a copy do produto deve tender a `SOS`

### Recaida

Usar:

- `Recaida`
- `recomeco`
- `registrar este momento`

Evitar:

- falha
- derrota
- voltar para zero

### Motivos

Usar:

- `motivos`

Uso secundario permitido:

- `ancoras`

Observacao:

- `ancoras` funciona melhor como imagem de apoio, nao como nome principal da funcionalidade

### Gatilhos

Usar:

- `gatilhos`
- `situacoes que te deixam mais vulneravel`

Evitar:

- mapa de risco como nome principal

Observacao:

- em onboarding e check-in, o mais claro e apresentar primeiro a ideia e depois o termo

### Fissura

Usar como termo principal:

- `vontade`
- `impulso`

Uso secundario permitido:

- `fissura`

Observacao:

- `fissura` e compreensivel para parte dos usuarios, mas nao deve carregar sozinho a tela toda

### Bloqueador

Usar:

- `Bloqueador`
- `protecao`
- `protecao ativa`

Evitar:

- VPN local
- DNS
- infraestrutura

Observacao:

- termos tecnicos podem aparecer em suporte, nao como copy-base da tela

### Analytics

Usar:

- `Analytics`
- `evolucao`
- `historico`
- `padroes`

Evitar:

- inteligencia do sistema
- reprogramacao do cerebro
- score milagroso

## Padrao para metas

Falar de metas como:

- `meta inicial`
- `primeira meta`
- `proxima meta`

Evitar:

- desafio infantilizado
- meta arbitraria sem contexto

## Padrao para CTAs

CTAs principais devem:

- deixar claro o proximo passo
- usar verbo direto
- evitar abstracao

Exemplos de direcao:

- registrar meu dia
- continuar
- ir para o SOS
- salvar e recomecar
- ver minha evolucao

## Termos banidos de produto

- prototipo
- legado
- shell
- sync manual
- status tecnico
- regra do produto
- debug
- hidratacao do estado
